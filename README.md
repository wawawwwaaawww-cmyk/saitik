# Наркологическая служба "Свой Нарколог"

Статический маркетинговый сайт для частной наркологической службы. Сайт построен на ванильном HTML5, CSS и JavaScript без использования сборщиков и бэкенда.

## Структура проекта

```
/
├── index.html                 # Главная страница
├── about.html                 # О клинике
├── contacts.html              # Контакты
├── vyvod-iz-zapoya.html      # Вывод из запоя
├── snyatie-lomki.html        # Снятие ломки
├── kodirovanie.html          # Кодирование
├── snyatie-pohmelya.html     # Снятие похмелья
├── kapelnitsa-ot-pohmelya.html # Капельница от похмелья
├── style.css                  # Основные стили
├── script.js                  # Общий JavaScript
├── assets/                    # Статические ресурсы
│   ├── icons/                 # Иконки
│   ├── otzyv/                 # Изображения отзывов
│   └── *.jpg, *.png, *.svg    # Изображения и иконки
├── vercel.json               # Конфигурация Vercel
└── README.md                 # Этот файл
```

## Развертывание на Vercel

### Автоматический деплой через GitHub

1. **Подключите репозиторий к Vercel:**
   - Зайдите в [Vercel Dashboard](https://vercel.com/dashboard)
   - Нажмите "Add New..." → "Project"
   - Импортируйте GitHub репозиторий с сайтом

2. **Настройте проект:**
   - Vercel автоматически определит тип проекта как "Static Site"
   - Убедитесь, что настройки соответствуют:
     - **Framework Preset**: Other
     - **Root Directory**: `./` (корень репозитория)
     - **Build Command**: оставьте пустым (для статического сайта не нужна сборка)
     - **Output Directory**: оставьте пустым (файлы уже в корне)

3. **Переменные окружения (если нужны):**
   - Добавьте необходимые переменные в разделе "Environment Variables"
   - Например: `ANALYTICS_ID`, `API_ENDPOINT` и т.д.

4. **Развертывание:**
   - Нажмите "Deploy"
   - Первый деплой займет несколько минут
   - После успешного деплоя сайт будет доступен по URL вида `https://your-project-name.vercel.app`

### Кастомный домен

1. В настройках проекта Vercel перейдите в раздел "Domains"
2. Добавьте свой домен (например, `your-site.ru`)
3. Следуйте инструкциям Vercel для настройки DNS:
   - Добавьте CNAME запись `www` → `cname.vercel-dns.com`
   - Добавьте A запись `@` → `76.76.19.61` (или используйте CNAME)

### Конфигурация маршрутизации

Файл `vercel.json` содержит настройки для корректной работы многостраничного сайта:

- **Чистые URL**: страницы доступны как по `/about.html`, так и по `/about`
- **Статические ресурсы**: все файлы из папки `assets/` корректно обслуживаются
- **CSS и JS**: файлы `style.css` и `script.js` доступны в корне
- **Главная страница**: `/` перенаправляет на `index.html`

## Локальная разработка

Для локальной разработки сайта используйте любой статический сервер:

### С помощью Python
```bash
python3 -m http.server 8000
# Откройте http://localhost:8000
```

### С помощью Node.js (serve)
```bash
npx serve .
# Откройте http://localhost:3000
```

### С помощью PHP
```bash
php -S localhost:8000
# Откройте http://localhost:8000
```

## Особенности проекта

- **Многостраничная архитектура**: каждая страница отдельный HTML файл
- **Общие ресурсы**: `style.css` и `script.js` используются на всех страницах
- **Адаптивный дизайн**: корректно работает на мобильных устройствах
- **Интерактивные элементы**: модальные окна, слайдеры, анимации
- **SEO-оптимизация**: семантическая HTML5 разметка, мета-теги

## Поддержка

При возникновении проблем с деплоем:

1. Проверьте логи сборки в Vercel Dashboard
2. Убедитесь, что все файлы присутствуют в репозитории
3. Проверьте корректность путей в `vercel.json`
4. Проверьте, что нет конфликтующих файлов в `public/` папке (если она существует)

## Автоматический деплой

После настройки Vercel будет автоматически развертывать сайт при:
- Пуше в основную ветку (main/master)
- Создании Pull Request
- Публикации нового релиза

Для отключения автоматического деплоя измените настройки в Vercel Dashboard → Settings → Git.
